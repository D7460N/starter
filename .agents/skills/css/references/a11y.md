# Accessibility

Focus-visible outlines. `accent-color` for native controls. `::selection` and `::marker` matched to brand. Visually-hidden text for screen-reader-only content.

## Only `:focus-visible` for focus rings

```css
@layer state {
  :focus-visible {
    outline: max(0.0625rem, 0.15em) solid var(--accent);
    outline-offset: 0.25em;
  }

  a:focus-visible {
    outline-width: 0.25em;
    outline-color: currentColor;
  }
}
```

`:focus-visible` activates only when the browser determines focus came from keyboard navigation, not mouse click. Mouse users don't see distracting outlines on click; keyboard users see clear focus.

## Only this for the accent color

```css
@layer themes {
  :root {
    accent-color: var(--accent);
  }
}
```

Native checkboxes, radios, range sliders, progress bars, and (where supported) text-selection caret color follow the accent. No control replacement needed.

## Selection and marker colors

```css
@layer themes {
  ::selection {
    background-color: var(--accent);
  }

  ::marker {
    color: var(--accent);
  }
}
```

`::selection` styles the text-selection background. `::marker` styles `<li>` bullets and `<details>` triangles.

## Visually hidden content

```css
@layer state {
  .visually-hidden:not(:focus):not(:active) {
    position: absolute;
    inline-size: 0.0625rem;
    block-size: 0.0625rem;
    overflow: hidden;
    clip-path: inset(50%);
    white-space: nowrap;
  }
}
```

Content that screen readers must announce but sighted users should not see (skip-links, form-error summaries). When the element receives focus or is activated, it appears — for keyboard users navigating skip-links.

(Note: the `.visually-hidden` class is shown for portability. In this architecture's HTML, replace the class selector with a semantic descendant or attribute selector.)

## Scrollbar coloring

```css
@layer themes {
  :root {
    scrollbar-color: var(--accent) Canvas;
  }
}
```

`Canvas` is the system canvas color (matches `color-scheme`). The first value is the thumb, the second is the track.

## Reduced motion

Respected in the `transitions` reference. Forms part of the accessibility posture.

## What a11y never does

- Never `:focus` (use `:focus-visible`)
- Never remove focus styles entirely (`outline: none` without a replacement is a violation)
- Never rely on color alone to convey state (combine with shape, position, text)
- Never substitute custom JS focus management for native focus
